# Benchmark Reference API — Frozen Contract (v1)

Status: **FROZEN** as of 2026-07-16. Every reference implementation under
`benchmarks/reference-apis/` MUST conform to this document exactly. Any change
to this file is a contract version bump and requires re-validating all
implementations. Source audit: `docs/analysis/language-bench-audit.md`.

This spec exists because the 17 implementations had drifted into three
incompatible contract families (audit §1) — the same endpoint returned
different shapes and performed different work per language, making
cross-language rankings meaningless.

## 0. Canonical shape family: Family C

The canonical contract is **family C** — the shapes implemented by
`crates/networker-endpoint/src/routes.rs` and its C# port
`src/Networker.Endpoint/Endpoints.cs` — extended with the orchestrator
transport contract (`/download/{size}`, `/health` with `runtime`).

Why family C and not B (the scripting-language majority):

1. `networker-endpoint` is the shipped product baseline: every VM provisioned
   by `install.sh` runs it, and it is the reference the orchestrator
   (`benchmarks/orchestrator`) validates against. The benchmark suite's job is
   to compare languages against *this* server's workload.
2. The shared dataset's `transform_input0` checksum has only ever matched the
   family C transform contract (`generate-bench-data.py`).
3. The C# migration keeps `networker-endpoint`'s port (`src/Networker.Endpoint`)
   at byte-parity with the Rust routes; family C is the only family with two
   independently maintained, test-covered implementations.

The orchestrator contract (`benchmarks/orchestrator/src/validator.rs`,
`deployer.rs`) is layered on top: `/health` requires `runtime` + `version`,
and downloads use the path form `/download/{size}`.

## 1. Transport, ports, auth, headers

- **Ports**: HTTPS + HTTP/2 on `8443` (env `BENCH_PORT` overrides). TLS certs
  are read from `$BENCH_CERT_DIR/cert.pem` + `key.pem`
  (default `BENCH_CERT_DIR=/opt/bench`).
- **Auth**: if env `BENCH_API_TOKEN` is set, every route except `/health`
  requires `Authorization: Bearer <token>`; otherwise auth is disabled.
  Rejection: HTTP 401, body `{"error":"unauthorized"}`.
- **Benchmark headers** — every `/api/*` response MUST carry:
  - `Server-Timing: app;dur=<ms>` (one decimal, server-side handler time)
  - `Cache-Control: no-store, no-cache, must-revalidate`
  - `Timing-Allow-Origin: *`
  - `Access-Control-Allow-Origin: *`
- **Errors**: JSON `{"error":"<message>"}` with the status codes given per
  endpoint below. No HTML error pages.
- JSON responses use `Content-Type: application/json`. Field *names and types*
  are normative; key order and whitespace are not (checksums use canonical
  JSON, §7).

## 2. Shared dataset (bench-data.json) — REQUIRED, load failure is FATAL

All data served by the `/api/*` endpoints comes from
`benchmarks/reference-apis/shared/bench-data.json`, generated by
`generate-bench-data.py` (fixed seed, committed to the repo, deployed to every
VM at `/opt/bench/bench-data.json` by `install.sh`).

**Resolution order** (first existing path wins):

1. `$BENCH_DATA_PATH` (if the env var is set)
2. `/opt/bench/bench-data.json`
3. `../shared/bench-data.json` relative to the server's source directory
   (repo-checkout convenience; equivalently
   `benchmarks/reference-apis/shared/bench-data.json` from the repo root)

**Failure policy (P0#2)**: dataset load failure MUST be fatal at startup —
print the reason to stderr and exit non-zero. There is **no PRNG fallback**:
if `BENCH_DATA_PATH` is set, that exact file must exist and parse; if it is
unset, one of the fallback paths must exist and parse. After parsing, the
implementation MUST verify the counts below and exit non-zero on mismatch.
Silent fallback is what made C# benchmark different input data than PHP
(audit F2).

*Sole exception*: `networker-endpoint` (Rust + its C# port) is a dual-role
production diagnostic server; when **no** dataset file exists anywhere and
`BENCH_DATA_PATH` is unset it may fall back to PRNG data for its non-benchmark
duties — but an explicitly configured `BENCH_DATA_PATH` that fails to load is
fatal there too, and benchmark runs always deploy the dataset.

**Schema** (`_version: 2`):

| Field | Type | Count | Element shape |
|---|---|---|---|
| `_version` | int | — | `2` |
| `users` | array | 100 | `{id:int, name:str, email:str, score:float(2dp), created_at:str(RFC3339)}` |
| `search_corpus` | array | 1000 | `str` (`"word-word-N"`) |
| `timeseries` | array | 10000 | `{ts:int, value:float(4dp), category:str}` — **objects, not bare floats** |
| `transform_inputs` | array | 10 | `{seed:int, fields:[str], values:[int]}` |
| `expected_checksums` | object | 4 keys | see §7 |

Users have **no** `age`, `active`, or `department` fields. Timeseries entries
are **objects** with `value` — implementations that summed the raw entries
(Python TypeError→500, Node NaN) or unmarshalled them into `float64` (Go →
zeros) must read the `value` field (see §9 porting notes).

## 3. Worker / process policy (P0#5)

Policy: **every implementation must be able to saturate all logical CPUs,
using its idiomatic production scaling mechanism, and the worker count must be
controlled by one knob**: env **`BENCH_WORKERS`** (default = logical CPU
count). Runs MUST record `nproc` and the effective `BENCH_WORKERS` next to
results.

| Language | Mechanism | `BENCH_WORKERS` maps to | Default |
|---|---|---|---|
| Rust (networker-endpoint) | tokio multi-thread runtime | worker threads | cores |
| Go | goroutines / scheduler | `GOMAXPROCS` | cores |
| C# (all variants) | Kestrel + ThreadPool | (in-process scheduler; knob advisory, document `nproc`) | cores |
| C++ | asio `io_context` thread pool | pool threads | cores |
| Java | `HttpServer` executor | fixed thread-pool size | cores |
| Node.js | `cluster` | worker processes | cores |
| Python | uvicorn | `--workers` processes | cores |
| Ruby | puma | `workers` processes (threads 5:5 per worker) | cores |
| PHP | Swoole | `worker_num` | cores (**not** the current hard-coded 4) |
| nginx | `worker_processes` | worker processes | `auto` |

Single-core comparisons set `BENCH_WORKERS=1` everywhere. A run mixing
defaulted and pinned implementations is invalid.

## 5. Endpoint contracts

### 5.1 `GET /health`

Constant-work: the response body MUST be a byte-constant string precomputed at
process startup — no per-request timestamp formatting, allocation-heavy
serialization, or dataset access. (Previously Java served a static string
while Rust formatted RFC3339 per request — different work in the one workload
CI measured weekly.)

Required fields:

```json
{"status":"ok","runtime":"<impl id>","version":"<impl or runtime version>"}
```

- `status`: literally `"ok"`.
- `runtime`: non-empty string identifying the implementation; use the
  reference directory name (`go`, `python`, `nodejs`, `java`, `ruby`, `php`,
  `cpp`, `csharp-net8`, …, `nginx`) or `rust` / `dotnet` for the
  networker-endpoint pair. A `/<version>` suffix is allowed.
- `version`: non-empty string (implementation or runtime version).

Additional constant fields are permitted (networker-endpoint adds
`"service"`). Auth-exempt. Status 200 only.

### 5.2 `GET /download/{size}`

- `{size}`: decimal byte count in `[0, 2147483648]` (2 GiB). Values above the
  cap are clamped. Non-integer → 400.
- Body: exactly `size` bytes, every byte `0x42` (`'B'`), streamed in
  **8192-byte chunks** (fill byte and chunk size are part of the measured
  workload — audit F10; canonical values are the 15-implementation majority).
- Headers: `Content-Length: <size>`, `Content-Type: application/octet-stream`,
  `X-Download-Bytes: <size>`, `Server-Timing: proc;dur=<ms>`.
- `GET /download?bytes=N` (default 1024) is a **deprecated alias** kept on
  networker-endpoint for pre-0.28.28 testers; new code and all reference
  implementations use the path form. The orchestrator validates
  `/download/1024` and `/download/65536` byte-exactly.

### 5.3 `POST /upload`

Drain the request body without buffering it wholesale; respond:

```json
{"received_bytes":<int>}
```

(`timestamp` may be included.) Headers: `X-Networker-Received-Bytes: <int>`,
`Server-Timing: recv;dur=<ms>`; echo `X-Networker-Request-Id` if present.

### 5.4 `GET /api/users?page=N&sort=<field>&order=<asc|desc>`

- `page`: int, default 1, minimum clamp 1. Selects the 100-user window
  `users[(page-1)*100 .. page*100)` of the dataset (the dataset has 100 users,
  so page 1 is the whole set and page ≥ 2 yields `[]` with status 200).
- `sort`: one of `id | name | email | score | created_at`; default and any
  unrecognized value → `id`. String fields compare bytewise (ordinal); `score`
  compares as float64. Sort MUST be stable (dataset order breaks ties).
- `order`: `desc` reverses; anything else (incl. default) is ascending.
- Response: **bare JSON array** (no wrapper object) of the first **20** users
  of the sorted window, each user object exactly as stored in the dataset:

```json
[{"id":1,"name":"Steve Moore","email":"steve.moore123@bench.dev","score":57.61,"created_at":"2026-05-11T08:30:00Z"}, …]
```

### 5.5 `POST /api/transform`

Request body (Content-Type: application/json):

```json
{"seed":1,"fields":["network","cache"],"values":[173,3315,190]}
```

All three fields optional; defaults `seed=0`, `fields=[]`, `values=[]`.
Syntactically invalid JSON → 400.

Response — SHA-256-hex (lowercase) of each field string's UTF-8 bytes, and
`values` reversed (elements passed through unmodified):

```json
{"seed":1,"hashed_fields":["<64-hex>","<64-hex>"],"reversed_values":[190,3315,173]}
```

### 5.6 `GET /api/aggregate[?range=start,end]`

- `range` is accepted for compatibility and **ignored** (reserved): the stats
  are always computed over the full 10,000-point `timeseries`. It is never
  required (family A's 400-on-missing-range is non-conformant).
- Algorithm (normative, float64 throughout):
  1. `values` = `timeseries[i].value` in dataset order; sort ascending.
  2. `mean` = sequential sum of the **sorted** values ÷ n.
  3. `p50` = `values[floor(n*0.50)]`, `p95` = `values[floor(n*0.95)]`,
     `max` = last element. (Index = float multiply then truncate.)
  4. Quintiles: `chunk = floor(n/5)`; for i in 0..5 the slice
     `values[i*chunk .. (i+1)*chunk)` yields
     `{category:"q<i+1>", count:chunk, mean:r2(sum/chunk), min:r2(first), max:r2(last)}`.
  5. `r2(x) = floor(x*100 + 0.5) / 100` (round half away from zero to 2
     decimals; equivalent for the positive values in this dataset).
- Response:

```json
{"total_points":10000,"mean":50.11,"p50":50.55,"p95":76.62,"max":91.83,
 "categories":[{"category":"q1","count":2000,"mean":31.31,"min":21.28,"max":37.55}, …]}
```

(Numbers above are illustrative; the exact canonical response for the frozen
dataset is pinned by `expected_checksums.aggregate_default`, §7.)

### 5.7 `GET /api/search?q=<term>&limit=N`

- `q`: default `"test"`. Treated as a **regex** (case-sensitive); if the
  pattern fails to compile, fall back to literal substring search. (Languages
  whose regex engine differs on exotic patterns are exercised only with
  literal-safe patterns by the validator: `network`, `test`.)
- `limit`: int, default 20, values > 100 clamped to 100.
- Scan `search_corpus` in order; for each item record the byte offset of the
  first match. Sort matches by `(position asc, item asc bytewise)`; take
  `limit`.
- Response (`total_matches` counts **all** matches, before truncation):

```json
{"query":"network","total_matches":57,"returned":10,
 "results":[{"rank":1,"item":"network-cache-42","match_position":0}, …]}
```

`rank` is 1-based after sorting.

### 5.8 `POST /api/upload/process`

Read the whole body (any content type), then compute:

- `crc32`: IEEE CRC-32 of the body, 8 lowercase hex chars, zero-padded.
- `sha256`: SHA-256 of the body, 64 lowercase hex chars.
- **zlib** (RFC 1950, with header/adler — *not* raw deflate, *not* gzip) at
  **level 6** (zlib default). Report `compressed_size`.

```json
{"original_size":39,"compressed_size":47,"crc32":"7f53f2a1","sha256":"<64-hex>"}
```

`original_size`, `crc32`, `sha256` MUST be byte-exact across languages.
`compressed_size` must come from zlib level 6 but may vary by ±small amounts
across zlib library versions — validators compare it with tolerance, not
equality. (Go's `flate` and C#'s `DeflateStream` raw-deflate variants are
non-conformant: different bytes *and* different CPU work.)

### 5.9 `GET /api/delayed?ms=N&work=<ignored>`

- `ms`: int, default 10, **clamped to [1, 100]** (Python's unclamped sleep was
  a DoS — audit drift table).
- The sleep MUST be an async/timer-based delay that does not block a worker
  thread (Swoole coroutine sleep, Beast timer, etc.) — otherwise this endpoint
  measures thread starvation, not the runtime.
- `work` is reserved: accepted and ignored.

```json
{"requested_ms":10,"actual_ms":10.42}
```

### 5.10 `GET /api/validate?seed=N`

- `seed`: int, default 42, echoed back verbatim (it does not affect output).
- Returns the dataset's `expected_checksums` object:

```json
{"seed":42,"checksums":{
  "users_page1":"<64-hex>","aggregate_default":"<64-hex>",
  "search_network_top10":"<64-hex>","transform_input0":"<64-hex>"}}
```

v1 semantics: the server **echoes** the checksums from its loaded dataset —
this proves which dataset the server loaded, and the external validator does
the real verification by fetching the four canonical requests (§7) and
hashing them itself. (Server-side recomputation is the planned P1 upgrade,
audit C9.)

Optional. If present, any HTML is fine; it is never measured or validated.

## 7. Checksums — canonical JSON and the four pinned requests

`expected_checksums` pins the canonical response of four requests against the
frozen dataset. Definition:

- **Canonical JSON**: UTF-8, object keys sorted lexicographically (byte
  order), no whitespace (separators `,` and `:`), floats in shortest
  round-trip decimal form (Python `repr`/ECMA-262 semantics; ints without
  `.0`).
- **Checksum** = lowercase SHA-256 hex of the canonical JSON of the parsed
  response body.

| Key | Request |
|---|---|
| `users_page1` | `GET /api/users?page=1&sort=name&order=asc` |
| `aggregate_default` | `GET /api/aggregate` |
| `search_network_top10` | `GET /api/search?q=network&limit=10` |
| `transform_input0` | `POST /api/transform` with body `transform_inputs[0]` from the dataset |

A validator verifies a server by: fetch → parse JSON → re-serialize
canonically → SHA-256 → compare with `expected_checksums` from
`bench-data.json`. `benchmarks/validate/run-validation.sh` implements exactly
this. `generate-bench-data.py` computes the expected values by replicating
the §5 algorithms — if you change an algorithm you must regenerate the
dataset and bump `_version`.

## 8. C# runtime ladder

All nine C# variants (`csharp-net48`, `net6`, `net7`, `net8`, `net8-aot`,
`net9`, `net9-aot`, `net10`, `net10-aot`) are **kept**. In wave 2 they are
regenerated from **one template** with version-gated deltas only (TFM, AOT
publish flags, H3 availability), so that "only the runtime differs" actually
holds (audit F13). Do not hand-edit individual variants.

## 9. Wave-2 porting notes (per language)

Every implementation must be ported to §5 exactly, must delete its PRNG
fallback (§2), and must implement `BENCH_WORKERS` (§3). Known deltas found by
the audit, per language:

- **Go** (`go/main.go`): users struct declares `score` as int and timeseries
  as bare floats — both silently corrupt the real dataset; use
  `{Id int; Name, Email string; Score float64; CreatedAt string}` and
  `{Ts int; Value float64; Category string}` and make unmarshal errors fatal.
  Stop re-unmarshalling per request (parse once at startup). Replace `flate`
  with zlib (`compress/zlib`, level 6). Family A response shapes → family C.
  Aggregate: drop the `range`-required 400 and the `i%5` bucketing.
- **C# reference variants** (`csharp-*/Program.cs`): `GetProperty("age")`
  and `GetDouble()` on the int-typed `score` throw → silent PRNG fallback;
  parse the §2 schema and fail fatally. Replace hand-rolled StringBuilder
  JSON with System.Text.Json. `DeflateStream` → `ZLibStream` level 6
  (`CompressionLevel` mapping must yield zlib 6). Validate response needs the
  `{seed, checksums}` wrapper. Regenerate all variants from one template (§8).
- **Node** (`nodejs/server.js`): aggregate sums timeseries objects → NaN; use
  `p.value`. Family B wrappers (`{page,…,users}`, `stats` nesting,
  `total_matches` before truncation is already correct) → family C shapes.
  Cluster workers = `BENCH_WORKERS`.
- **Python** (`python/server.py`): aggregate `sum(dicts)` TypeError → 500; use
  `p["value"]`. Clamp `ms` (≤100) and `limit` (≤100). Family B → C. Decide the
  uvicorn/hypercorn identity (audit F12): whichever is chosen must be used in
  Dockerfile, deploy.sh, and CI alike, with access logs off; drop the Alt-Svc
  h3 advertisement if uvicorn stays.
- **Ruby** (`ruby/config.ru`): family B → C; `total_matches` before
  truncation; puma `workers ENV["BENCH_WORKERS"]`.
- **PHP** (`php/server.php`): family B → C; `worker_num` from `BENCH_WORKERS`;
  coroutine sleep instead of `usleep`.
- **C++** (`cpp/server.cpp`): family B → C; timer-based delay instead of
  `sleep_for` on io_context threads.
- **Java** (`java/Server.java`): replace the hand-written JSON scanner (breaks
  on escaped quotes) with a real JSON approach; family A/B mix → C; `/health`
  gains `version` (done in wave 1); zeros/64 KiB download → 0x42/8 KiB.
- **nginx**: baseline for transport only; serves `/health`, `/download`,
  `/upload`. It never implements `/api/*` — the capability matrix (audit C5)
  must exclude it from apibench.
- **Rust + C# networker-endpoint**: already canonical (wave 1 added
  `/download/{size}`, `runtime` in `/health`, 0x42/8 KiB fill, fatal
  `BENCH_DATA_PATH`).

## 10. Conformance checklist (machine-checkable)

1. `GET /health` → 200, JSON with `status=="ok"`, non-empty `runtime`,
   `version`; two requests return byte-identical bodies.
2. `GET /download/1024` → exactly 1024 bytes, all `0x42`,
   `Content-Length: 1024`.
3. The four §7 requests hash to `expected_checksums`.
4. `GET /api/delayed?ms=600000` responds in ≲ 200 ms (clamp works).
5. `GET /api/users?page=999` → 200 `[]`.
6. `POST /api/transform` with body `not-json` → 400.
7. All `/api/*` responses carry the four §1 benchmark headers.
8. With `BENCH_DATA_PATH=/nonexistent`, the process exits non-zero at startup.
9. With `BENCH_API_TOKEN` set, `GET /api/users` without the header → 401;
   `/health` still 200.
