# Impeccable Design Context

## Design Context

### Users
- **Primary**: Network/IT engineers and DevOps/SRE teams
- **Technical level**: High — comfortable with CLI, network protocols, raw data
- **Context**: Diagnosing network performance issues, comparing cloud providers, validating infrastructure. Often under time pressure during incidents or deployment validation
- **Job to be done**: Get clear, trustworthy network diagnostics data quickly — both in real-time (dashboard) and as sharable artifacts (HTML reports)

### Brand Personality
**Technical, precise, reliable.**
- Earns trust through accuracy and clarity, not flashiness
- Feels like a tool built by engineers for engineers
- Data is the hero — the interface serves the data, not the other way around

### Aesthetic Direction
- **Visual tone**: Terminal/hacker aesthetic — lean into monospace typography, dark theme, data-dense layouts. Like a sophisticated terminal UI that happens to run in a browser
- **References**: Grafana, Datadog (professional observability dashboards), Warp/Fig (modern terminal tools — monospace but polished, premium CLI feel)
- **Anti-references**: Generic SaaS dashboards with gradient hero sections, marketing-site aesthetics, overly decorative UI with low information density
- **Theme**: Dark mode primary (`#0a0b0f` base). Light mode for HTML reports (print-friendly)
- **Brand colors**: Purple `#863bff` (logo), Cyan `#47bfff` / `cyan-400` (primary accent), deep navy backgrounds
- **Typography**: Monospace-first (`Cascadia Code`, `JetBrains Mono`, `Consolas`). System sans-serif for HTML reports

### Design Principles
1. **Data density over decoration** — Every pixel should communicate information. Prefer compact, scannable layouts over spacious "breathing room" that wastes screen real estate
2. **Terminal confidence** — The interface should feel like a powerful instrument panel. Monospace type, precise alignment, status-coded colors. Users should feel in control
3. **Trust through consistency** — Status colors (green/red/yellow/cyan) must be semantically consistent everywhere. Numbers must be precise. Labels must be unambiguous
4. **Progressive disclosure** — Show summary first, let users drill into detail. Collapsible sections, expandable rows, linked detail pages
5. **Zero chrome** — No gradients, no shadows, no rounded-everything. Flat surfaces, thin borders, color-coded accents. Let the data be the visual interest
